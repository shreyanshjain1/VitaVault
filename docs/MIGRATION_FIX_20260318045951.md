# Migration Fix: 20260318045951_init

This patch replaces `prisma/migrations/20260318045951_init/migration.sql` with a safe no-op migration.

## Why

The latest VitaVault `0_init` migration already contains the device/mobile sync schema. The older `20260318045951_init` migration was still trying to create the same enums/tables again, causing Prisma shadow database validation to fail with:

```txt
ERROR: type "ReadingSource" already exists
```

## What changed

The migration file is kept for history compatibility, but the duplicate SQL statements were removed.

## Verify

```powershell
npx prisma migrate dev
npx prisma generate
npm run lint
npm run typecheck
npm run test:run
```
